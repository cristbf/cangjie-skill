# 阶段 1.5 — 三重验证筛选

## 目标

从候选单元池里,筛出**真正值得做成独立 skill 的方法论单元**。通不过的降级为 example / 引用 / 术语,但不独立成 skill。

这是 book2skill 区别于"书摘工具"的核心质量门。

## 三重验证 (全部通过才录取)

### V1 — 跨域验证 (Cross-domain)

**问题**: 这个单元在书中**至少 2 个独立的语境**下有佐证吗?

- "独立"的含义: 不是同一个案例换个说法,而是两个不同的故事/不同的章节/不同的对象都在讲同一个道理
- **通过**: 《穷查理宝典》中"逆向思维"在投资决策、避免灾难、教学方法三个独立场景中都出现 → 通过
- **不通过**: 某个漂亮句子只在一章里出现过一次,没有书内的独立证据 → 降级为金句 example

**为什么**: 在多个语境中反复出现的,才是作者真正想传达的稳定方法论,而不是一时兴起的表达。

**B 类（叙事型）调整**: 跨域验证的标准从"2 个独立段落"改为"2 个独立事件/时期/案例"。叙事型书籍的方法论不是作者用语言明确表达的,而是隐含在多个事件中的行为模式 — 因此"独立语境"指的是"不同的历史事件/不同的人生阶段/不同的决策场景"。

### V2 — 预测力测试 (Predictive Power)

**问题**: 能用这个单元,推导出书里没明说的某个问题的答案吗?

#### 场景来源（禁止自行设计）

V2 的测试场景**不能由提取者自己设计**（有循环论证风险）。必须从以下两个来源获取:

1. **用户提供的真实问题**: 在阶段 0 时顺带收集用户"我最近遇到的一个具体问题",作为 V2 的测试场景
2. **对抗性场景**: 由一个独立的 adversarial agent 设计故意刁难的场景,核心问题是:"这个方法论在什么情况下会彻底失效?"

如果两个来源都不可用,才能 fallback 到自行设计,但必须记录 `scenario_source: self-designed` 并在审计中标记为低置信度。

#### 评判标准（三维度）

答案是否有意义,不再凭感觉判断,而是检查三个维度:

| 维度 | 问题 | 通过标准 |
|------|------|---------|
| **可复现** | 换一个人按同样步骤,能得出类似结论吗? | 推导过程是明确的,不是靠直觉跳跃 |
| **可证伪** | 这个结论有没有办法被检验或反驳? | 结论是具体的,不是"见机行事"之类的万能回答 |
| **非平凡** | 这个结论是不是"努力就会成功"之类的废话? | 结论提供了新信息,不是重复前提 |

三个维度全部通过才算 V2 通过。

- **通过**: 推导可复现 + 结论可证伪 + 非平凡 → 通过
- **不通过**: 任一维度不通过 → 这个单元没有真正的解释力,降级

**为什么**: 真正的方法论必须有**外推能力**。如果它只能复述书里的例子,它是描述不是方法。同时,场景设计与评判必须分离,避免自我验证。

**C 类（理论型）调整**: 预测力测试改为"能否用这个推理路径分析一个新的论题"。评判标准调整为:
- **可复现**: 换一个人按同样推理路径,能得出类似结论吗?
- **可外推**: 这个推理路径能否应用于原书未讨论的领域?
- **非平凡**: 推理路径提供了新的理解方式,不只是"换个说法"?
- 场景来源: 由 adversarial agent 提供一个新的论题,测试推理路径的迁移能力。

### V3 — 独特性检验 (Exclusivity)

**问题**: 这个单元的回答和同类书的典型回答有显著差异吗?

#### 前置步骤: V3 基线生成

在执行 V3 之前,先运行 `extractors/v3-baseline-extractor.md`,为每个候选单元生成对照基线:

- 选择该领域 **3 本最有影响力的代表性著作**
- 基于训练集知识,生成这些书对同一问题的典型回答
- 输出到 `candidates/v3-baselines.md`,包含 `baseline_verdict`: unique / overlap / insufficient

#### V3 判断逻辑

以基线提取器的产出为锚点,分三种情况:

| baseline_verdict | 含义 | V3 结论 |
|-----------------|------|---------|
| `unique` | 同类书的回答与本书有**显著路径差异** | ✅ 通过 |
| `overlap` | 同类书的回答与本书**基本一致** | ⚠️ 需人工复核 |
| `insufficient` | 同类书没有直接讨论过这个问题 | 退回 fallback 判断 |

**fallback 判断**（仅当 baseline 为 insufficient 时使用）:

- 如果把作者名字抹掉,一个对书本领域毫无了解的聪明人也能说出来 → 不通过
- 必须是作者**独特视角 / 反直觉见解 / 独特术语体系** → 通过
- **通过**: 段永平的"stop doing list" — 主动列出不做什么,反常识 → 通过
- **不通过**: "要尊重时间" — 太常识了,没人需要一个 skill 来告诉自己这个

**通过示例**: 《穷查理宝典》的"逆向思维"
- 同类书（《思考快与慢》《原则》《清单革命》）的典型回答是"识别偏差 → 建立正向纠正机制"
- 芒格的独特路径是"不问怎么成功,先问怎么死" — 把负面推理置于正面推理之上
- → 路径差异显著,V3 通过

**不通过示例**: "要尊重时间的力量"
- 同类书几乎都会提到这个观点,没有任何路径差异
- → V3 不通过,降级为金句 example

**为什么**: 常识不需要 skill 承载,Claude 本身就知道。只有作者的**差异化见解**才值得固化成 skill。但"差异化"必须有参照物 — 基线提取器提供了这个参照物。

## 验证执行流程

1. 把阶段 1 的 5 个 candidates/*.md 合并成一个总候选池
2. 去重: 同一方法论被多个 extractor 提取的,合并成一条
3. **预处理**: 运行 `extractors/v3-baseline-extractor.md`,为每条候选生成 V3 对照基线,写入 `candidates/v3-baselines.md`
4. 对每条候选跑 V1 / V2 / V3,记录判断和理由
   - V1: 跨域验证 — 书中至少 2 个独立段落有佐证
   - V2: 预测力测试 — 场景来源(用户问题 / 对抗性场景) + 三维度评判(可复现 / 可证伪 / 非平凡)
   - V3: 独特性检验 — 以基线提取器产出为锚点判断路径差异
5. 通过的,写入 `books/<slug>/verified.md`,进入阶段 2
6. 未通过的,写入 `books/<slug>/rejected/<id>.md`,**必须写明不通过的是哪一项、原因是什么** (审计价值)

## 常见失败模式

1. **V1 作弊** — 把同一例子换个说法算两处。要求: 必须是不同章节 + 不同对象 + 不同结论。
2. **V2 作弊** — 用一个其实书里讨论过的类似问题冒充"新问题"。要求: 新问题应该让人第一眼不知道书里怎么说。
3. **V2 自我验证** — 提取者自己设计场景、自己评判答案。要求: 场景必须来自用户问题或对抗性 agent,不能自行设计。
4. **V2 虚假外推** — 答案看似新颖但实际不可复现或不可证伪。要求: 检查三个维度(可复现 / 可证伪 / 非平凡)。
5. **V3 过松** — 只要"说得比较文雅"就认为不是常识。要求: 以基线提取器的产出为锚点,看**内容路径**是否与同类书有差异。
6. **V3 基线编造** — V3 基线引用的同类书不存在或内容是编造的。要求: 基线书必须是该领域真实有影响力的著作。

## 数量预期

经验上,一本方法论密集的书 (如《穷查理宝典》) 通过率约 30–50%。一本散文类书可能只有 5–10%。通过率过低 (<5%) 或过高 (>80%) 都要警惕:
- 过低: extractor 可能质量差,要重跑
- 过高: 验证标准可能太松
